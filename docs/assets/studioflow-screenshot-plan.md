# StudioFlow Screenshot Plan

## Hero Screenshot Recommendation

Use the dashboard or booking calendar as the hero image.

### Best choice

- Dashboard with today’s appointments, revenue snapshot, notifications, and recent activity

### Why

- It communicates “real SaaS product” fastest
- It shows multiple data types in one calm, premium screen
- It works well for both portfolio and GitHub

## Thumbnail Recommendation

Use the booking calendar with a visible appointment card and top-shell context.

### Why

- It is the most recognizable operational workflow
- It gives a stronger product signal than a generic dashboard card grid
- It works well in smaller preview sizes

## What Each Screenshot Should Show

### Dashboard

- Show today’s appointments, revenue, unread notifications, and recent activity
- Use meaningful seeded data with balanced density

### Calendar

- Show multiple staff columns and strong appointment density
- Keep filters calm and readable

### Appointment Detail

- Show service, staff, payment, notes, communications, and activity history
- Use one strong appointment with believable context

### Clients / Staff / Services

- Show realistic names, titles, and service details
- Avoid sparse or placeholder-looking data

### Payments

- Show a mix of pending and paid states
- Keep one strong payment record visible

### Consent Forms

- Show both template and submission status context if possible

### Notifications

- Open the notification panel with a visible unread badge

### Audit Logs

- Show actor, action, entity, and timestamp in a clean table or feed

### Public Booking

- Show either the service/staff/time steps or the confirmation state
- Use branded, believable customer-facing context

### Manage Booking

- Show lookup success plus reschedule or cancel confirmation

### Analytics

- Show charts and summaries with meaningful non-empty data

## Screens That Must Use Polished Seeded Data

- Dashboard
- Calendar
- Appointment detail
- Payments
- Notifications
- Audit logs
- Analytics
- Public booking confirmation

## Consistency Notes

- Prefer the same light theme, browser chrome, and viewport width across the whole set
- Avoid mixing local and hosted environments in one screenshot sequence
- Use consistent seeded names and business context throughout
- Keep the browser zoom at 100%
- Crop consistently with enough shell context to show this is a real application

## Cropping Notes

- Portfolio hero: wider crop with shell and page heading visible
- GitHub screenshots: prioritize readable product content over browser chrome
- LinkedIn: use tighter crops that emphasize the most distinctive UI region

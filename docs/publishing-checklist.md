# StudioFlow Publishing Checklist

## Product Ready

- Core internal workflows behave correctly.
- Public booking and customer self-service flows are working.
- Hosted demo account is verified.
- First-run onboarding behavior is understood and intentional.
- No obvious placeholder data or broken states remain on hero screens.

## Screenshots Ready

- Hero screenshot selected.
- Portfolio screenshot order is finalized.
- GitHub screenshot order is finalized.
- LinkedIn screenshot order is finalized.
- All captured screens use polished seeded data.
- Cropping and theme consistency are checked.

## Demo Ready

- Backend is awake before recording.
- Demo account credentials are verified.
- Recording path is practiced once without speaking.
- Short-form and long-form demo orders are finalized.
- One strong appointment, payment, notification, and booking record are prepared.
- Public booking and manage-booking flows are ready to show cleanly.

## GitHub Ready

- README is current and professional.
- Setup and deployment notes are understandable.
- Demo and screenshot placeholders are ready to fill with final assets.
- Repo subtitle, summary, and feature highlights are polished.
- No accidental secrets or temporary notes are left in tracked files.

## Portfolio Ready

- Project page copy is finalized.
- Project card copy is finalized.
- Hero screenshot is selected.
- Live demo, GitHub, and case study links are ready.
- Case study sections align with the product story used in interviews.

## Resume Ready

- One-line and two-line summaries are finalized.
- Role-specific bullets are prepared for full-stack, backend, frontend, and SaaS/product roles.
- ATS-friendly wording is ready.
- Technical stack line is current.

## LinkedIn Ready

- Launch post is ready.
- Short showcase post is ready.
- Architecture-focused post is ready.
- “What I learned” post is ready.
- Screenshot order or carousel outline is selected.

## Recruiter Share Ready

- Recruiter outreach message is ready.
- LinkedIn connection note is ready.
- Project share message with links is ready.
- Follow-up after interview message is ready.

## Interview Prep Ready

- Strongest 5 and 10 talking points are ready.
- Architecture explanation is ready.
- Scaling and future-improvement answer is ready.
- Product and business value explanation is ready.
- “What I personally built” answer is ready.

## Final Pre-Publish Pass

- Live demo link works.
- GitHub repo link works.
- Portfolio links work.
- Hosted login note reflects the current hosted setup.
- The project story is consistent across README, portfolio, resume, and LinkedIn.

# Screenshot And Asset Plan

Use this plan to capture StudioFlow consistently across README, portfolio, and LinkedIn.

## Recommended Hero Image

### Hero Screenshot

- Screen:
  dashboard or calendar with a visible location switcher and meaningful live data
- What should be visible:
  app shell, premium layout, strong page heading, operational content, believable seeded data
- Why it matters:
  this gives the fastest impression that StudioFlow is a real multi-surface SaaS product
- Suggested filename:
  `01-studioflow-hero-dashboard.png`

## Capture Tips

- Use the seeded demo data, not ad-hoc manual records.
- Capture screens with balanced content, not overly empty or overloaded states.
- Keep browser chrome minimal where possible.
- Prefer one consistent viewport size for portfolio shots.
- Use the dashboard or calendar as the hero/thumbnail in most contexts.
